# Glossary of Commands

This is an equivalent / translator to Linear.app concepts

## Glossary

Epic = Project inside a Team
Task = Issue
Bug = Issue
Issue = Issue

## Structure of the Items

- When an Issue has a "design" keyword, the Technical Implementation and Deployment Notes sections should only state `N/A`
- Create "team labels" to their corresponding issue type instead of using `[label]` at the beginning of the issues's title
- Along the "team labels", add the corresponding "type label" among (create any if not present):
  - Feature
  - Bug
  - Improvement
  - Technical Debt
  - Chore
  - Spike
  - Architectures
- Create any template that you consider necessary
